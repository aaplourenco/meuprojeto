Introdução ao Git e GitHub

Passo 1: O que é Git e por que usá-lo

O Git é um sistema de controle de versão distribuído que ajuda a gerenciar o código-fonte ao longo do tempo. Ele permite que vários desenvolvedores trabalhem no mesmo projeto simultaneamente, rastreando todas as mudanças e facilitando a colaboração.

Passo 2: Instalação e Configuração do Git

Para instalar e configurar o Git, siga os passos abaixo:

Instalação

No Windows:

Baixe e instale o Git do site oficial: https://git-scm.com/

No macOS:

brew install git

No Linux:

sudo apt-get install git

Configuração Inicial

Após instalar o Git, configure seu nome de usuário e e-mail:

git config --global user.name "Seu Nome"
git config --global user.email "seu.email@example.com"

Passo 3: Comandos Básicos do Git

Aprenda os comandos básicos do Git para começar a usar o controle de versão:

Inicializar um repositório Git:

git init

Adicionar arquivos ao índice (staging area):

git add .

Registrar as mudanças no repositório:

git commit -m "Mensagem do commit"

Enviar as mudanças para um repositório remoto:

git push

Atualizar o repositório local com mudanças do remoto:

git pull

Passo 4: Criando e Gerenciando Repositórios no GitHub

Siga os passos para criar um novo repositório no GitHub e conectá-lo ao seu repositório local:

Criando um Repositório no GitHub

  • Acesse GitHub e faça login na sua conta.
  • Clique no botão New (Novo) para criar um novo repositório.
  • Dê um nome para o repositório e clique em Create repository (Criar repositório).

Conectando o Repositório Local ao GitHub

Adicione o repositório remoto no GitHub ao seu repositório local:

git remote add origin git@github.com:usuario/nome-do-repositorio.git

Passo 5: Como Colaborar com Outros Desenvolvedores Usando Pull Requests e Branches

Criando e Mudando para Novas Branches

Para criar e mudar para uma nova branch:

git checkout -b nome-da-branch

Fazendo Pull Requests

Após fazer commits na nova branch, envie as mudanças para o repositório remoto:

git push origin nome-da-branch

No GitHub, abra um pull request da sua branch para a branch principal do projeto.

Revisando e Mesclando Pull Requests

Revise o pull request, discuta as mudanças e, se estiver tudo certo, mescle o pull request na branch principal.
